title: Automatic Case Grouping
description: How multiple reports about the same issue are automatically clustered into cases
---

Quejas Ciudadanas automatically groups related citizen reports into **cases** based on geographic proximity and problem type. This prevents duplicate processing and provides visibility into the scale of each infrastructure issue.

## What is a Case?

A **case** represents a single infrastructure problem that may have been reported by multiple citizens. Instead of processing each report individually, the system:

1. Identifies reports about the same issue
2. Groups them into a single case
3. Assigns a unique case ID
4. Tracks the number of reports per case

<Info>
When citizens search for reports, they see cases (grouped reports), not individual reports. This provides a clearer picture of widespread issues.
</Info>

## How Cases are Created

Cases are automatically created when a report is submitted. The grouping logic is handled by the database stored procedure `sp_guardar_report_ciudadanos`.

### Grouping Criteria

Reports are grouped into the same case when they share:

1. **Similar Location**: Matching or very similar geohash values
2. **Same Problem Type**: Identical problem classification (Tipo de Problema)
3. **Same City**: Located in the same municipality

<Tip>
Geohash is the key technology that enables automatic clustering. Reports within approximately 2.4 meters of each other will have the same 9-character geohash.
</Tip>

## Case Data Structure

Each case contains:

```csharp
public class CasoFiltrado
{
    public int Id_caso { get; set; }           // Unique case ID
    public string latitud { get; set; }        // Representative latitude
    public string longitud { get; set; }       // Representative longitude
    public string Problema { get; set; }       // Problem category name
    public string Clase_problema { get; set; } // Problem class name
    public string Tipo_problema { get; set; }  // Problem type name
    public string Departamento { get; set; }   // Department name
    public string Ciudad { get; set; }         // City name
    public DateTime Fecha_registro { get; set; } // First report date
    public int Estado { get; set; }            // Case status (1=Pending, 2=In Progress, 3=Completed)
    public int Num_quejas { get; set; }        // Number of reports in this case
}
```

## Case Querying

Government agents and citizens can search for cases using various filters:

```csharp
public async Task<IActionResult> consultar_reportes([FromForm] Filtros_buscar_reportes model)
{
    if (string.IsNullOrEmpty(model.selectDepto))
        model.selectDepto = "0000";
    
    if (string.IsNullOrEmpty(model.selectCiudad))
        model.selectCiudad = "0000";
    
    List<CasoFiltrado> consulta_repor = await _context.ObtenerCasosFiltradosAsync(model);
    
    var data = consulta_repor.Select(caso => new
    {
        id = caso.Id_caso,
        lat = caso.latitud,
        lon = caso.longitud,
        problema = caso.Problema,
        clase = caso.Clase_problema,
        tipo = caso.Tipo_problema,
        departamento = caso.Departamento,
        ciudad = caso.Ciudad,
        fecha = caso.Fecha_registro,
        estado = caso.Estado,
        cantidad = caso.Num_quejas
    }).ToList();
    
    return Json(data);
}
```

### Filter Options

</Note>

## Map Visualization

On the search page, cases are displayed as cluster markers on the map:

```javascript
function crearMarcador(item) {
    const icon = L.divIcon({
        html: `<div style="background:${estadoColores[item.estado]};color:white;padding:4px 6px;border-radius:15px;border:2px solid #fff;font-size:12px;box-shadow:0 0 3px rgba(0,0,0,0.5);transform:translate(-50%,-50%)">${item.cantidad}</div>`,
        className: '',
        iconSize: [30, 30],
        iconAnchor: [15, 15]
    });
    const marker = L.marker([item.lat, item.lon], { icon }).addTo(map)
        .bindTooltip(item.titulo)
        .on('click', () => mostrarAlerta(item.id, 0));
    marcadores[item.id] = marker;
}
```

Each marker displays:
- **Color**: Indicates case status (Blue=Pending, Yellow=In Progress, Red=Completed)
- **Number**: Shows count of reports in the case
- **Tooltip**: Problem description on hover
- **Click Action**: Opens detailed case view

<Tip>
The marker color and count provide immediate visual feedback about problem severity and status.
</Tip>

## Viewing Case Details

When a user clicks on a case marker or table row, they see detailed information including:

### Primary Report Information

The case shows details from one representative report:

```javascript
function mostrarAlerta(id_caso, id_report) {
    var formData = new FormData();
    formData.append("IdCaso", id_caso);
    formData.append("IdReporte", id_report);
    
    $.ajax({
        url: '/Reportes/consultar_caso_x_report',
        method: 'POST',
        data: formData,
        contentType: false,
        processData: false,
        success: function (response) {
            actualizarCarta(response.info[0]);
            generarCarrusel(response.fotos);
            mostrarreportes_relacionados(response.info[0].id_caso);
        }
    });
}
```

## Benefits of Case Grouping

<Steps>
  <Step title="Eliminates Duplicate Work">
    Government agents can address multiple reports with a single response instead of handling each one individually.
  </Step>
  
  <Step title="Shows Problem Severity">
    The count of reports per case immediately indicates which issues are affecting the most citizens.
  </Step>
  
  <Step title="Provides Context">
    Seeing multiple perspectives on the same issue helps agents understand the full scope of the problem.
  </Step>
  
  <Step title="Improves Efficiency">
    Resources can be prioritized based on the number of citizens affected.
  </Step>
</Steps>

| 2 | En proceso | Yellow (#FFD700) | Under investigation by agent |

## Technical Implementation

The case grouping is primarily handled at the database level:

```csharp
public async Task<List<CasoFiltrado>> ObtenerCasosFiltradosAsync(Filtros_buscar_reportes model)
{
    var parametros = new[]
    {
        new SqlParameter("@problemaSelect", model.problemaSelect),
        new SqlParameter("@claseProblemaSelect", model.claseProblemaSelect),
        new SqlParameter("@tipoProblemaSelect", model.tipoProblemaSelect),
        new SqlParameter("@selectDepto", model.selectDepto),
        new SqlParameter("@selectCiudad", model.selectCiudad),
        new SqlParameter("@Estado_del_caso", model.Estado_del_caso)
    };
    
    var result = await Consulta_report
        .FromSqlRaw("EXEC sp_ConsultaCasosFiltrados @problemaSelect, @claseProblemaSelect, @tipoProblemaSelect, @selectDepto, @selectCiudad, @Estado_del_caso", parametros)
        .ToListAsync();
    
    return result;
}
```

The stored procedure `sp_ConsultaCasosFiltrados` performs the complex grouping logic, likely using geohash and problem type to identify related reports and aggregate them into cases.

<Note>
The database-level grouping ensures consistency and performance when dealing with large numbers of reports.
</Note>
