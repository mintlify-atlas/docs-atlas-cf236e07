## Overview

Report models handle the creation, querying, and management of citizen reports and infrastructure cases. These models support the full lifecycle of reports from submission to agent responses.

## Reporte_nuevo

Model for creating a new report.

```csharp
public class Reporte_nuevo
{
    public int Id_user { get; set; }
    public int id_estado { get; set; }
    public string Latitud { get; set; }
    public string Longitud { get; set; }
    public string Geohash { get; set; }
    public string TipoProblema { get; set; }
    public string Ciudad { get; set; }
    public string Direccion { get; set; }
    public string Descripcion { get; set; }
}
```

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `Id_user` | `int` | User ID creating the report |
| `id_estado` | `int` | Initial state ID |
| `Latitud` | `string` | Latitude coordinate |
| `Longitud` | `string` | Longitude coordinate |
| `Geohash` | `string` | Geohash for location clustering |
| `TipoProblema` | `string` | Problem type identifier |
| `Ciudad` | `string` | City name |
| `Direccion` | `string` | Street address |
| `Descripcion` | `string` | Problem description |

### Properties

| Property | Type | Description |

| Property | Type | Description |
|----------|------|-------------|

## Relationships

- Reports are grouped into cases based on geohash proximity
- `Reporte_nuevo.Id_user` references `Usuario_Logueado.Id_user`
- `Reporte_nuevo.TipoProblema` references `Tipo_Problema` catalog
- `Reporte_nuevo_fotos.Id_reporte` references report ID
- Case filters reference catalog models for problems, locations, and states
