## Overview

Quejas Ciudadanas uses SQL Server as its database backend with Entity Framework Core for ORM and stored procedures for complex operations. This guide will walk you through setting up the complete database infrastructure.

## Prerequisites

<CardGroup cols={2}>
  <Card title="SQL Server" icon="database">
    SQL Server 2016 or later installed and running
  </Card>
  <Card title="Configuration" icon="gear">
    Connection string configured in appsettings.json
  </Card>
</CardGroup>

<Note>
  Ensure you have completed the [Installation](/installation) and [Configuration](/configuration) steps before proceeding.
</Note>

## Database Creation

<Steps>
  <Step title="Connect to SQL Server">
    Open SQL Server Management Studio (SSMS) or Azure Data Studio and connect to your SQL Server instance.

    <CodeGroup>
    ```bash SSMS
    # Connect using:
    Server: localhost
    Authentication: SQL Server Authentication
    Login: sa
    Password: [Your SA Password]
    ```

    ```bash sqlcmd
    sqlcmd -S localhost -U sa -P YourPassword
    ```
    </CodeGroup>
  </Step>

  <Step title="Create Database">
    Execute the following SQL to create the database:

    ```sql
    CREATE DATABASE quejas_ciudadana;
    GO

    USE quejas_ciudadana;
    GO
    ```
  </Step>

  <Step title="Verify Database Creation">
    ```sql
    SELECT name FROM sys.databases WHERE name = 'quejas_ciudadana';
    ```

    You should see `quejas_ciudadana` in the results.
  </Step>
</Steps>

## Stored Procedures

The application uses stored procedures for critical operations. Below are the required stored procedures:

### User Authentication

<CodeGroup>
```sql sp_validar_login
CREATE PROCEDURE sp_validar_login
    @Identificacion NVARCHAR(50),
    @Clave NVARCHAR(255)
AS
BEGIN
    SET NOCOUNT ON;

    SELECT 
        u.Id_usuario,
        u.Nombre,
        u.Apellido,
        u.Correo,
        u.Id_tipo_user,
        t.Descripcion AS TipoUsuario
    FROM Usuarios u
    INNER JOIN Tipo_Usuario t ON u.Id_tipo_user = t.Id_tipo_user
    WHERE u.Identificacion_reg = @Identificacion 
        AND u.CLAVE = @Clave
        AND u.Activo = 1;
END;
GO
```

```sql sp_registrar_usuario
CREATE PROCEDURE sp_registrar_usuario
    @Id_tipo_user INT,
    @Id_identificacion INT,
    @Identificacion_reg NVARCHAR(50),
    @Nombre NVARCHAR(100),
    @Apellido NVARCHAR(100),
    @Correo NVARCHAR(150),
    @FechaNacimiento DATE,
    @Id_genero INT,
    @CLAVE NVARCHAR(255)
AS
BEGIN
    SET NOCOUNT ON;
    
    BEGIN TRY
        BEGIN TRANSACTION;
        
        -- Check if user already exists
        IF EXISTS (SELECT 1 FROM Usuarios WHERE Identificacion_reg = @Identificacion_reg)
        BEGIN
            SELECT 0 AS Resultado, 'El usuario ya existe' AS Mensaje;
            ROLLBACK TRANSACTION;
            RETURN;
        END
        
        -- Insert new user
        INSERT INTO Usuarios (
            Id_tipo_user, Id_identificacion, Identificacion_reg,
            Nombre, Apellido, Correo, FechaNacimiento, Id_genero, CLAVE, Activo
        )
        VALUES (
            @Id_tipo_user, @Id_identificacion, @Identificacion_reg,
            @Nombre, @Apellido, @Correo, @FechaNacimiento, @Id_genero, @CLAVE, 1
        );
        
        SELECT 1 AS Resultado, 'Usuario registrado exitosamente' AS Mensaje;
        
        COMMIT TRANSACTION;
    END TRY
    BEGIN CATCH
        IF @@TRANCOUNT > 0
            ROLLBACK TRANSACTION;
            
        SELECT 0 AS Resultado, ERROR_MESSAGE() AS Mensaje;
    END CATCH
END;
GO
```
</CodeGroup>

### Report Management

<CodeGroup>
```sql sp_guardar_report_ciudadanos
CREATE PROCEDURE sp_guardar_report_ciudadanos
    @Id_user INT,
    @id_estado INT,
    @Latitud DECIMAL(10, 8),
    @Longitud DECIMAL(11, 8),
    @Geohash NVARCHAR(50),
    @TipoProblema INT,
    @Ciudad SMALLINT,
    @Direccion NVARCHAR(500),
    @Descripcion NVARCHAR(MAX)
AS
BEGIN
    SET NOCOUNT ON;
    
    BEGIN TRY
        BEGIN TRANSACTION;
        
        DECLARE @Id_reporte INT;
        
        -- Insert report
        INSERT INTO Reportes (
            Id_usuario, Id_estado, Latitud, Longitud, Geohash,
            Id_tipo_problema, Id_ciudad, Direccion, Descripcion,
            Fecha_reporte
        )
        VALUES (
            @Id_user, @id_estado, @Latitud, @Longitud, @Geohash,
            @TipoProblema, @Ciudad, @Direccion, @Descripcion,
            GETDATE()
        );
        
        SET @Id_reporte = SCOPE_IDENTITY();
        
        SELECT 
            1 AS Resultado, 
            'Reporte creado exitosamente' AS Mensaje,
            @Id_reporte AS Id_reporte;
        
        COMMIT TRANSACTION;
    END TRY
    BEGIN CATCH
        IF @@TRANCOUNT > 0
            ROLLBACK TRANSACTION;
            
        SELECT 0 AS Resultado, ERROR_MESSAGE() AS Mensaje;
    END CATCH
END;
GO
```

### Query Procedures

<CodeGroup>
```sql sp_ConsultaCasosFiltrados
CREATE PROCEDURE sp_ConsultaCasosFiltrados
    @problemaSelect INT,
    @claseProblemaSelect INT,
    @tipoProblemaSelect INT,
    @selectDepto SMALLINT = NULL,
    @selectCiudad SMALLINT = NULL,
    @Estado_del_caso INT
AS
BEGIN
    SET NOCOUNT ON;
    
    SELECT 
        r.Id_reporte,
        r.Id_caso,
        r.Direccion,
        r.Descripcion,
        r.Fecha_reporte,
        r.Latitud,
        r.Longitud,
        p.Nombre AS Problema,
        cp.Nombre AS ClaseProblema,
        tp.Nombre AS TipoProblema,
        c.A002DESCRIPCION AS Ciudad,
        d.A001DESCRIPCION AS Departamento,
        e.Descripcion AS Estado,
        u.Nombre + ' ' + u.Apellido AS Reportante
    FROM Reportes r
    INNER JOIN Tipo_Problema tp ON r.Id_tipo_problema = tp.Id_Tipo_Problema
    INNER JOIN Clase_Problema cp ON tp.Id_Clase_Problema = cp.Id_Clase_Problema
    INNER JOIN Problema p ON cp.Id_Problema = p.Id_Problema
    INNER JOIN CAT002_CIUDADES c ON r.Id_ciudad = c.A002ID
    INNER JOIN CAT001_DEPTOS d ON c.A002CODDEPTO = d.A001CODIGO
    INNER JOIN Estado_caso e ON r.Id_estado = e.Id_estado_caso
    INNER JOIN Usuarios u ON r.Id_usuario = u.Id_usuario
    WHERE 
        (@problemaSelect = 0 OR p.Id_Problema = @problemaSelect)
        AND (@claseProblemaSelect = 0 OR cp.Id_Clase_Problema = @claseProblemaSelect)
        AND (@tipoProblemaSelect = 0 OR tp.Id_Tipo_Problema = @tipoProblemaSelect)
        AND (@selectDepto IS NULL OR d.A001ID = @selectDepto)
        AND (@selectCiudad IS NULL OR c.A002ID = @selectCiudad)
        AND (@Estado_del_caso = 0 OR r.Id_estado = @Estado_del_caso)
    ORDER BY r.Fecha_reporte DESC;
END;
GO
```

### Agent Response Procedures

<CodeGroup>
```sql sp_Guardar_RespuestaAgente
CREATE PROCEDURE sp_Guardar_RespuestaAgente
    @Id_caso INT,
    @id_opcion_caso INT,
    @Descripcion NVARCHAR(MAX)
AS
BEGIN
    SET NOCOUNT ON;
    
    BEGIN TRY
        BEGIN TRANSACTION;
        
        DECLARE @Id_respuesta INT;
        
        -- Insert agent response
        INSERT INTO Respuestas_Agente (
            Id_caso, Id_opcion_respuesta, Descripcion, Fecha_respuesta
        )
        VALUES (
            @Id_caso, @id_opcion_caso, @Descripcion, GETDATE()
        );
        
        SET @Id_respuesta = SCOPE_IDENTITY();
        
        -- Update case status if needed
        UPDATE Reportes 
        SET Id_estado = @id_opcion_caso
        WHERE Id_caso = @Id_caso;
        
        SELECT 
            1 AS Resultado, 
            'Respuesta guardada exitosamente' AS Mensaje,
            @Id_respuesta AS Id_respuesta;
        
        COMMIT TRANSACTION;
    END TRY
    BEGIN CATCH
        IF @@TRANCOUNT > 0
            ROLLBACK TRANSACTION;
            
        SELECT 0 AS Resultado, ERROR_MESSAGE() AS Mensaje;
    END CATCH
END;
GO
```

CREATE PROCEDURE Sp_obtener_respuestas_agente
    @IdCaso INT
AS
BEGIN
    SET NOCOUNT ON;
    
    SELECT 
        ra.Id_respuesta,
        ra.Id_caso,
        ra.Descripcion,
        ra.Fecha_respuesta,
        e.Descripcion AS EstadoRespuesta,
        a.Nombre + ' ' + a.Apellido AS NombreAgente
    FROM Respuestas_Agente ra

    LEFT JOIN Usuarios a ON ra.Id_agente = a.Id_usuario
    WHERE ra.Id_caso = @IdCaso
    ORDER BY ra.Fecha_respuesta DESC;
END;
GO

## Apply Schema with Entity Framework

Alternatively, you can use Entity Framework Core migrations to create the database schema:

<Steps>
  <Step title="Create Initial Migration">
    ```bash
    dotnet ef migrations add InitialCreate
    ```
  </Step>

  <Step title="Update Database">
    ```bash
    dotnet ef database update
    ```

    This will create all tables defined in the `DbContext`.
  </Step>

  <Step title="Verify Tables">
    ```sql
    USE quejas_ciudadana;
    SELECT TABLE_NAME 
    FROM INFORMATION_SCHEMA.TABLES 
    WHERE TABLE_TYPE = 'BASE TABLE'
    ORDER BY TABLE_NAME;
    ```
  </Step>
</Steps>

<Warning>
  Entity Framework migrations will create tables but **not** stored procedures. You must create stored procedures manually using the SQL scripts above.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Cannot Create Database">
    **Error**: `CREATE DATABASE permission denied`
    
    **Solution**:
    - Ensure you're logged in as `sa` or a user with `dbcreator` role
    - Grant permission: `ALTER SERVER ROLE dbcreator ADD MEMBER [YourUser]`
  </Accordion>

  <Accordion title="Stored Procedure Fails">
    **Error**: `Could not find stored procedure 'sp_validar_login'`
    
    **Solution**:
    - Verify you're connected to the correct database: `USE quejas_ciudadana`
    - Check stored procedure exists: `SELECT * FROM sys.procedures WHERE name = 'sp_validar_login'`
    - Re-run the CREATE PROCEDURE script
  </Accordion>

  <Accordion title="Foreign Key Constraint Errors">
    **Error**: `FOREIGN KEY constraint failed`
    
    **Solution**:
    - Ensure parent tables are created before child tables
    - Verify referenced data exists before inserting
    - Check the relationship definitions in the schema
  </Accordion>

  <Accordion title="Entity Framework Migration Fails">
    **Error**: `An error occurred while applying migration`
    
    **Solution**:
    ```bash
    # Remove last migration
    dotnet ef migrations remove
    
    # Drop database and recreate
    dotnet ef database drop
    dotnet ef database update
    ```
  </Accordion>
</AccordionGroup>
