## Controllers

Controllers handle HTTP requests, process user input, interact with models, and return appropriate views or data.

#### Login Flow Example

<CodeGroup>
```csharp Login Action (HomeController.cs:28-53)
[HttpPost]
public async Task<IActionResult> inicio_secion(inicia_secion model)
{
    Usuario_Logueado usuario = await _context.ValidarLoginAsync(model);

    if (usuario != null)
    {
        // Store user information in session
        HttpContext.Session.SetInt32("Id_user", usuario.Id_user);
        HttpContext.Session.SetInt32("Id_tipo_user", usuario.Id_tipo_user);
        HttpContext.Session.SetString("TipoUsuario", usuario.TipoUsuario ?? "");
        HttpContext.Session.SetString("Nombre", usuario.Nombre ?? "");
        HttpContext.Session.SetString("Apellido", usuario.Apellido ?? "");
        HttpContext.Session.SetString("Correo", usuario.Correo ?? "");
        // ... more session variables

        return RedirectToAction("Index", "Reportes");
    }

    ViewBag.Error = "Usuario o contraseña incorrectos";
    return new RedirectToActionResult("Index", "Home", null);
}
```
</CodeGroup>

<Info>
Session data is stored server-side with a 30-minute idle timeout. The session ID is sent to the client as a cookie.
</Info>

#### Report Creation Example

<CodeGroup>
```csharp GuardarReporte Action (ReportesController.cs:74-135)
[HttpPost]
public async Task<IActionResult> GuardarReporte([FromForm]Reporte_nuevo model)
{
    var session = HttpContext.Session;
    var idUsuario = session.GetInt32("Id_user");
    model.Id_user = (int)idUsuario;
    model.id_estado = 1;

    // Save report to database via stored procedure
    respuesta_repote_registrado nuevo_reporte = 
        await _context.GuardarReporteCiudadanoAsync(model);
    
    if (nuevo_reporte != null)
    {
        // Process 6 image uploads
        var archivos = new List<IFormFile>();
        for (int i = 0; i < 6; i++)
        {
            var archivo = Request.Form.Files["Imagen" + i];
            if (archivo == null || archivo.Length == 0)
                return BadRequest($"Imagen {i + 1} no fue cargada.");
            archivos.Add(archivo);
        }

        // Create unique folder name
        var idReporte = $"{nuevo_reporte.Id_caso}_{nuevo_reporte.Id_reporte}_" +
                       $"{nuevo_reporte.Id_user_ciudadano}_{nuevo_reporte.llave}";

        // Save files and record paths in database
        string carpeta = Path.Combine("wwwroot/evidencias", idReporte.ToString());
        Directory.CreateDirectory(carpeta);

        // ... file processing logic

        return Ok(new { mensaje = "Reporte enviado correctamente" });
    }
    
    return BadRequest("Error al guardar el reporte.");
}
```
</CodeGroup>

<Note>
The report creation process involves:
1. Validating and saving report metadata via stored procedure
2. Creating a unique folder based on case/report/user IDs
3. Saving 6 required photos to the folder
4. Recording file paths in the database
</Note>

## Models

Models represent data structures and business entities. The application organizes models into three files:

### Catalog Models (clases_catalogo.cs)

Defines reference/lookup tables and their relationships.

**Location:** `Models/clases_catalogo.cs`

<CodeGroup>
```csharp Problem Hierarchy Models
public class Problema
{
    [Key]
    public int Id_Problema { get; set; }
    public string Nombre { get; set; }
    public ICollection<Clase_Problema> Clases_Problema { get; set; }
}

public class Clase_Problema

    public int Id_Clase_Problema { get; set; }

}
```
</CodeGroup>

<Info>
The problem hierarchy follows a three-level classification:
1. **Problema** (Problem) - e.g., "Infrastructure"
2. **Clase_Problema** (Problem Class) - e.g., "Roads"
3. **Tipo_Problema** (Problem Type) - e.g., "Pothole"
</Info>

## Custom Filters

### VerificaSesionAttribute

A custom action filter that enforces authentication.

**Location:** `Filtros/VerificaSesionAttribute.cs`

<CodeGroup>
```csharp VerificaSesionAttribute Implementation
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.Filters;

namespace quejas_ciudadanas.Filtros
{
    public class VerificaSesionAttribute : ActionFilterAttribute
    {
        public override void OnActionExecuting(ActionExecutingContext context)
        {
            var session = context.HttpContext.Session;
            var idUsuario = session.GetInt32("Id_user");

            if (idUsuario == null)
            {
                // User not logged in - redirect to home
                context.Result = new RedirectToActionResult(
                    "Index", "Home", null);
            }

            base.OnActionExecuting(context);
        }
    }
}
```
</CodeGroup>

#### Usage Examples

<CodeGroup>
```csharp Controller-Level Protection
[VerificaSesion]  // Protects all actions in controller
public class ReportesController : Controller
{
    // All actions require authentication
}
```

```csharp Action-Level Protection
public class HomeController : Controller
{
    public IActionResult Index()  // Public
    {
        return View();
    }

    [VerificaSesion]  // Protected action
    public IActionResult Privacy()
    {
        return View();
    }
}
```
</CodeGroup>

<Warning>
The `VerificaSesion` filter must be applied to any controller or action that requires authentication. Without it, unauthenticated users can access protected resources.
</Warning>
