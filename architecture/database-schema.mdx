## Database Overview

Quejas Ciudadanas uses **SQL Server** as its database management system, accessed through **Entity Framework Core 9.0.4**. The database stores user information, complaint reports, catalog data, and agent responses.

<CardGroup cols={2}>
  <Card title="Database Server" icon="server">
    SQL Server (Azure SQL Database in production)
  </Card>
  <Card title="ORM Framework" icon="code">
    Entity Framework Core 9.0.4 with Code-First and Database-First hybrid approach
  </Card>
  <Card title="Migration Strategy" icon="arrows-rotate">
    Stored procedures for complex operations, EF Core for CRUD
  </Card>
  <Card title="Connection Pooling" icon="layer-group">
    Automatic connection pooling via EF Core
  </Card>

## Entity Relationships

Relationships are configured in the `OnModelCreating` method:

<CodeGroup>
```csharp Entity Relationships (QuejasCiudadanasContext.cs:22-40)
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    // Problem -> Clase_Problema (One-to-Many)
    modelBuilder.Entity<Clase_Problema>()
        .HasOne(c => c.Problema)
        .WithMany(p => p.Clases_Problema)
        .HasForeignKey(c => c.Id_Problema);

    // Clase_Problema -> Tipo_Problema (One-to-Many)
    modelBuilder.Entity<Tipo_Problema>()
        .HasOne(t => t.Clase_Problema)
        .WithMany(c => c.Tipos_Problema)
        .HasForeignKey(t => t.Id_Clase_Problema);

    // Departamento -> Ciudad (One-to-Many)
    modelBuilder.Entity<CAT002_CIUDADES>()
        .HasOne(c => c.CAT001_DEPTOS)
        .WithMany(d => d.Ciudades)
        .HasForeignKey(c => c.A002CODDEPTO)
        .HasPrincipalKey(d => d.A001CODIGO);
}
```
</CodeGroup>

## Data Access Patterns

### Pattern 1: Direct Entity Framework Queries

Used for simple CRUD operations on catalog tables:

<CodeGroup>
```csharp Example: Fetching Dropdown Data
// Get all problem types
var problemas = _context.Problema.ToList();

// Get problem classes filtered by problem ID
var clases = _context.Clase_Problema
    .Where(c => c.Id_Problema == id)
    .Select(c => new { 
        id_Clase_Problema = c.Id_Clase_Problema, 
        nombre = c.Nombre 
    })
    .ToList();
```
</CodeGroup>

### Pattern 2: Stored Procedure Calls

Used for complex business logic and data transformations:

<CodeGroup>
```csharp Example: Calling Stored Procedure
// 1. Create parameters
var parametros = new[]
{
    new SqlParameter("@Id_user", userId),
    new SqlParameter("@Descripcion", description)
};

// 2. Execute with FromSqlRaw
var result = await _context.SomeDbSet
    .FromSqlRaw(
        "EXEC sp_stored_procedure_name @Id_user, @Descripcion", 
        parametros)
    .ToListAsync();

// 3. Return first result or entire list
return result.FirstOrDefault();
```
</CodeGroup>

<Warning>
**SQL Injection Prevention:** Always use `SqlParameter` objects when passing values to stored procedures. Never concatenate user input directly into SQL strings.
</Warning>

### Catalog Tables

<Accordion title="Problema, Clase_Problema, Tipo_Problema - Problem Classification">

```sql
-- Level 1: General problem category
CREATE TABLE Problema (
    Id_Problema INT PRIMARY KEY IDENTITY,
    Nombre VARCHAR(200)
);

-- Level 2: Problem class
CREATE TABLE Clase_Problema (
    Id_Clase_Problema INT PRIMARY KEY IDENTITY,
    Id_Problema INT FOREIGN KEY REFERENCES Problema(Id_Problema),
    Nombre VARCHAR(200)
);

-- Level 3: Specific problem type
CREATE TABLE Tipo_Problema (
    Id_Tipo_Problema INT PRIMARY KEY IDENTITY,
    Id_Clase_Problema INT FOREIGN KEY REFERENCES Clase_Problema(Id_Clase_Problema),
    Nombre VARCHAR(200)
);
```

**Example Hierarchy:**
- Problema: "Infraestructura"
  - Clase_Problema: "Vías"
    - Tipo_Problema: "Bache"
    - Tipo_Problema: "Grieta"
  - Clase_Problema: "Alumbrado"
    - Tipo_Problema: "Poste dañado"

</Accordion>

## Best Practices

<Steps>
  <Step title="Always Use Parameters">
    Never concatenate user input into SQL strings. Use `SqlParameter` objects.
  </Step>
  
  <Step title="Use Async Methods">
    Call `ToListAsync()`, `FirstOrDefaultAsync()`, etc. instead of synchronous versions.
  </Step>
  
  <Step title="Optimize Read-Only Queries">
    Add `.AsNoTracking()` to queries that won't modify entities.
  </Step>
  
  <Step title="Handle Nulls Gracefully">
    Use nullable types (`?`) and null-coalescing operators (`??`) when dealing with optional data.
  </Step>
  
  <Step title="Log SQL Exceptions">
    Wrap stored procedure calls in try-catch blocks and provide meaningful error messages.
  </Step>
</Steps>
