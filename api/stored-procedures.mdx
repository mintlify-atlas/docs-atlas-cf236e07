The Quejas Ciudadanas application uses SQL Server stored procedures for all database operations. This provides security, performance optimization, and separation of concerns.

<Info>
  All stored procedures are called via Entity Framework Core's `FromSqlRaw` method with parameterized queries to prevent SQL injection.
</Info>

## Authentication Procedures

### sp_validar_login

Validates user credentials and returns user information for session creation.

#### Parameters

<ParamField path="@Identificacion" type="string" required>
  User's identification number
</ParamField>

<ParamField path="@Clave" type="string" required>
  User's password (should be hashed)
</ParamField>

#### Returns

Single row with user information or no rows if authentication fails:

<ResponseField name="Id_user" type="int">
  User's unique identifier
</ResponseField>

<ResponseField name="Id_tipo_user" type="int">
  User type ID (1 = Citizen, 2 = Agent, etc.)
</ResponseField>

<ResponseField name="TipoUsuario" type="string">
  User type description
</ResponseField>

<ResponseField name="TipoIdentificacion" type="string">
  Identification document type
</ResponseField>

<ResponseField name="Identificacion" type="string">
  Identification number
</ResponseField>

<ResponseField name="Nombre" type="string">
  First name
</ResponseField>

<ResponseField name="Apellido" type="string">
  Last name
</ResponseField>

<ResponseField name="Correo" type="string">
  Email address
</ResponseField>

<ResponseField name="FechaNacimiento" type="DateTime">
  Date of birth
</ResponseField>

<ResponseField name="Genero" type="string">
  Gender description
</ResponseField>

#### Usage in Code

```csharp
var identificacionParam = new SqlParameter("@Identificacion", model.NumeroIdentificacion_login);
var claveParam = new SqlParameter("@Clave", model.Contrasena_login);

var result = await LoginTemp
    .FromSqlRaw("EXEC sp_validar_login @Identificacion, @Clave", 
                identificacionParam, claveParam)
    .ToListAsync();

return result.FirstOrDefault();
```

### sp_registrar_usuario

Creates a new user account in the system.

#### Parameters

<ParamField path="@Id_tipo_user" type="int" required>
  User type (1 = Citizen, 2 = Agent)
</ParamField>

<ParamField path="@Id_identificacion" type="string" required>
  Identification document type ID
</ParamField>

<ParamField path="@Identificacion_reg" type="int" required>
  Identification number
</ParamField>

<ParamField path="@Nombre" type="string" required>
  First name
</ParamField>

<ParamField path="@Apellido" type="string" required>
  Last name
</ParamField>

<ParamField path="@Correo" type="string" required>
  Email address
</ParamField>

<ParamField path="@FechaNacimiento" type="DateTime" required>
  Date of birth
</ParamField>

<ParamField path="@Id_genero" type="int" required>
  Gender ID
</ParamField>

<ParamField path="@CLAVE" type="string" required>
  Password (should be hashed before passing)
</ParamField>

#### Returns

<ResponseField name="NumeroIdentificacion_login" type="string">
  Registered identification number
</ResponseField>

<ResponseField name="Contrasena_login" type="string">
  Password for auto-login
</ResponseField>

<ResponseField name="Resultado" type="string">
  "1" if successful, "0" or error code if failed
</ResponseField>

#### Usage in Code

```csharp
var parametros = new[]
{
    new SqlParameter("@Id_tipo_user", model.tipo_usuario),
    new SqlParameter("@Id_identificacion", model.tipo_doc_regist),
    new SqlParameter("@Identificacion_reg", model.numero_doc_regist),
    new SqlParameter("@Nombre", model.nombre_regist),
    new SqlParameter("@Apellido", model.apellido_regist),
    new SqlParameter("@Correo", model.email_regist),
    new SqlParameter("@FechaNacimiento", model.fech_nacimien_regist),
    new SqlParameter("@Id_genero", model.sexo_regist),
    new SqlParameter("@CLAVE", model.clave_regist)
};

var result = await RegisterTemp
    .FromSqlRaw("EXEC sp_registrar_usuario @Id_tipo_user, @Id_identificacion, " +
                "@Identificacion_reg, @Nombre, @Apellido, @Correo, " +
                "@FechaNacimiento, @Id_genero, @CLAVE", parametros)
    .ToListAsync();
```

## Report Management Procedures

### sp_guardar_report_ciudadanos

Creates a new citizen report and associated case (or adds to existing case based on geohash proximity).

#### Parameters

<ParamField path="@Id_user" type="int" required>
  User ID from session
</ParamField>

<ParamField path="@id_estado" type="int" required>
  Initial case status (typically 1 for "New")
</ParamField>

<ParamField path="@Latitud" type="string" required>
  Geographic latitude coordinate
</ParamField>

<ParamField path="@Longitud" type="string" required>
  Geographic longitude coordinate
</ParamField>

<ParamField path="@Geohash" type="string" required>
  Geohash string for spatial grouping
</ParamField>

<ParamField path="@TipoProblema" type="string" required>
  Problem type ID
</ParamField>

<ParamField path="@Ciudad" type="string" required>
  City location code
</ParamField>

<ParamField path="@Direccion" type="string" required>
  Street address or description
</ParamField>

<ParamField path="@Descripcion" type="string" required>
  Detailed problem description
</ParamField>

#### Returns

<ResponseField name="Id_reporte" type="int">
  Newly created report ID
</ResponseField>

<ResponseField name="llave" type="string">
  Security key for report identification
</ResponseField>

<ResponseField name="Id_caso" type="int">
  Case ID (new or existing)
</ResponseField>

<ResponseField name="Id_user_ciudadano" type="int">
  Citizen user ID
</ResponseField>

#### Geohash Grouping Logic

The stored procedure uses geohash to automatically group similar reports:
- Reports with matching geohash and problem type are grouped into same case
- This creates clusters of related infrastructure issues in the same location
- Reduces duplicate work for government agents

## Best Practices

<Tip>
  **Parameterized Queries**: All stored procedure calls use `SqlParameter` objects to prevent SQL injection attacks.
</Tip>

<Warning>
  **Error Handling**: Wrap all database calls in try-catch blocks. The application throws generic exceptions that should be logged for debugging.
</Warning>

<Info>
  **Connection Management**: Entity Framework Core manages database connections automatically. No need to manually open/close connections.
</Info>

## Usage Example

Complete example of calling a stored procedure with EF Core:

```csharp
public async Task<List<CasoFiltrado>> ObtenerCasosFiltradosAsync(
    Filtros_buscar_reportes model)
{
    var parametros = new[]
    {
        new SqlParameter("@problemaSelect", model.problemaSelect),
        new SqlParameter("@claseProblemaSelect", model.claseProblemaSelect),
        new SqlParameter("@tipoProblemaSelect", model.tipoProblemaSelect),
        new SqlParameter("@selectDepto", model.selectDepto),
        new SqlParameter("@selectCiudad", model.selectCiudad),
        new SqlParameter("@Estado_del_caso", model.Estado_del_caso)
    };

    try
    {
        var result = await Consulta_report
            .FromSqlRaw(
                "EXEC sp_ConsultaCasosFiltrados @problemaSelect, " +
                "@claseProblemaSelect, @tipoProblemaSelect, " +
                "@selectDepto, @selectCiudad, @Estado_del_caso", 
                parametros)
            .ToListAsync();

        return result;
    }
    catch (Exception ex)
    {
        throw new Exception("Error al ejecutar el procedimiento almacenado", ex);
    }
}
```

## Performance Considerations

<Card title="AsNoTracking()" icon="gauge">
  Use `.AsNoTracking()` for read-only queries to improve performance:
  
  ```csharp
  var result = await ReporteInfraestructura
      .FromSqlRaw("EXEC Sp_consulta_reportes_unicos @IdCaso, @IdReporte", parametros)
      .AsNoTracking() // Improves performance for read-only data
      .ToListAsync();
  ```
</Card>

<Card title="Connection Pooling" icon="server">
  Configure connection pooling in `appsettings.json` for optimal database performance:
  
  ```json
  {
    "ConnectionStrings": {
      "DefaultConnection": "Server=...;Database=...;Min Pool Size=5;Max Pool Size=100;"
    }
  }
  ```
</Card>
